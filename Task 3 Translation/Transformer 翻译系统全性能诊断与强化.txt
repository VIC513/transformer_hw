Trae 重构指令：Transformer 翻译系统全性能诊断与强化（Harness Level）

角色设定：
你是资深算法工程师 + MLOps 工程师。你需要对现有代码库做“数据分布 + 模型监控 + 逻辑验证”的全方位重构，并交付一套可一键运行的训练与诊断体系。

Context（必须参考并复用现有结构/命名习惯）：
- debug_data.py
- generate_data.py
- train_mt.py
- 以及当前仓库的 text_processor.py / model.py （如有）

任务目标：
重写并整合一套高性能的“训练 + 诊断”体系，定位并缓解：
1) 词频偏见（例如“指生为医”）
2) 验证集过拟合（小数据集上训练过头）

具体重构要求：

1) 数据生成器强化（generate_data.py）
1.1 平衡度分析
- 数据生成结束后，自动统计并打印 subjects、occupations、foods 的分布直方图（纯文本 ASCII 柱状图即可）
- 输出需包含：总样本数、Top-N 项、长尾项数量
-柱状图需支持归一化显示。例如，出现频次最高的词占满 20 个字符宽度的 #，其他词按比例缩放。这能让我直观看到词频分布的‘长尾效应’。

1.2 逻辑解耦（Disjoint Combinations）
- 重构划分逻辑，确保训练集与验证集的“主谓宾组合”完全不重复
- 组合签名建议：template_id + (subject_id, verb_id, object_id/slot_id...)
- 要求：train 和 val 的组合签名集合严格互斥（disjoint）
- 需要输出：train/val 组合数、互斥性断言（如断言失败要直接报错）

1.3 脏数据清洗
- 增强 _dedup / 清洗逻辑：自动剔除由于模板嵌套或替换导致的畸形句子
- 至少包含：残留占位符、异常重复词/字符、非法符号混入、空串/极短串、明显语法崩坏的模式
- 清洗后输出：剔除数量、剔除原因统计 Top-K

2) 训练监控与日志系统（train_mt.py）
2.1 预测分布可视化（Entropy）
- 修改 translate_sentence：debug 模式下除 Top-5 Token 外，还要计算并显示 entropy（信息熵）
- 需要显示：step、top_tokens、top_probs、entropy
- 若熵过低且预测错误：输出明显警告（用于判断词频偏见）

2.2 黄金对比日志（对照训练集参考答案）
- 修改 append_translation_log：每条日志格式统一为：
  [EN原句] | [模型输出] | [训练集参考答案]
- 参考答案获取：从训练集检索 EN 完全匹配的目标句（或最接近的 1 条，需说明匹配策略）
- 高亮错误关键词：在“模型输出”中标注与参考答案不同的片段（用明显标记符即可，如 << >> ）
-由于训练时数据量可能较大，请在 train_mt.py 初始化时将训练集加载到内存 Hash Map 中，
以 EN_sentence 为 Key，实现  $O(1)$ 级别的参考答案检索，避免每轮日志都进行全表扫描。

2.3 Early Stopping
- 引入基于 val_loss 的早停机制
- 需要参数：patience、min_delta
- 需要输出：每次最佳 val_loss 更新记录、触发早停的原因与 epoch

3) 深度诊断工具（debug_data.py）
3.1 词表覆盖率检查（关键词低频预警）
- 自动对比 src_vocab 与 tgt_vocab（覆盖率/大小/UNK占比）
- 若 generate_data_4.py 中定义的关键 tokens（如“学生”等）在训练集中频率 < 5：必须触发警告并列出这些 tokens
- 输出需要可读：token、计数、是否进入 vocab、是否可能导致偏见/过拟合

3.2 推理压力测试（极端组合）
- 诊断结束前自动合成 5 组数据集中从未出现过的“极端组合”（更长、更复杂、组合未见）
极端组合应包含：
1) 超过 MAX_SEQ_LEN 的超长句；
2) 包含 generate_data.py 中从未出现过的未知字符（测试 UNK 处理）；
3) 语法正确但语义极其罕见的组合（测试模型是否真的理解结构）。
- 测试 processor.encode_src / encode_tgt 的稳定性：不崩溃、长度截断符合 max_seq_len、UNK 使用合理
- 输出：每条压力测试输入、编码长度、UNK 计数、是否触发异常
